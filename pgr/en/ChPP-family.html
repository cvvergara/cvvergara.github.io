Chinese Postman Problem - Family of functions¶

pgr_directedChPP_Cost.rst pgr_directedChPP.rst * pgr_directedChPP - Experimental - Get the aggregate cost of the shortest paths. * pgr_directedChPP_Cost - Experimental - Dijkstra’s algorithm for the shortest paths.

Given the following query:

pgr_directedChPP(\(sql\))

where \(sql = \{(id_i, source_i, target_i, cost_i, reverse\_cost_i)\}\)

and

  • \(source = \bigcup source_i\),
  • \(target = \bigcup target_i\),

The graphs are defined as follows:

Directed graph

The weighted directed graph, \(G(V,E)\), is definied by:

  • the set of vertices \(V\)
    • \(V = source \cup target \cup {start_{vid}} \cup {end_{vid}}\)
  • the set of edges \(E\)
    • \(E = \begin{cases} \text{ } \{(source_i, target_i, cost_i) \text{ when } cost >=0 \} & \quad \text{if } reverse\_cost = \varnothing \\ \text{ } \text{ } & \quad \text{ } \\ \text{ } \{(source_i, target_i, cost_i) \text{ when } cost >=0 \} & \quad \text{ } \\ \cup \{(target_i, source_i, reverse\_cost_i) \text{ when } reverse\_cost_i>=0 \} & \quad \text{if } reverse\_cost \neq \varnothing \\ \end{cases}\)

The problem

Given:

  • \(G(V,E)\)

Then:

\(pgr\_directedChPP(edges\_sql) = \boldsymbol{\Phi}\)

\(\boldsymbol{\Phi} = {(seq_i, node_i, edge\_i, cost_i, agg\_cost_i)}\)

Where:

  • \(seq_i = i\)
  • \(node_i \in V\)
  • \(edge_i = \begin{cases} id_{(node_i, node_{i+1},cost_i)} &\quad \text{when } i \neq | \pi | \\ -1 &\quad \text{when } i = | \pi | \\ \end{cases}\)
  • \(cost_i = cost_{(node_i, node_{i+1})}\)
  • \(agg\_cost_i = \begin{cases} 0 &\quad \text{when } i = 1 \\ \displaystyle\sum_{k=1}^{i} cost_{(node_{k-1}, node_k)} &\quad \text{when } i \neq 1 \\ \end{cases}\)
In other words: The algorithm is to find the shortest path which contains every edge in a graph and starts and ends on the same node, if it exists, in terms of a sequence of nodes and of edges,
  • \(seq\) indicates the relative position in the path of the \(node\) or \(edge\).
  • \(cost\) is the cost of the edge to be used to go to the next node.
  • \(agg\_cost\) is the cost from the start node up to the node.

If there is no path, the resulting will be no path found error.
