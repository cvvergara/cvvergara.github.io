deegree¶

Über¶

Die deegree Webservices sind Implementierungen der Webservice Spezifikationen des Open Geospatial Consortium (OGC) und der INSPIRE Network Services. deegree webservices 3.3 enthält die folgenden Dienste:

  • Web Feature Service (WFS): Ermöglicht den direkten Zugriff Vektordaten aus räumlichen Datenobjekten.
  • Web Map Service (WMS): Liefert Karten, die aus räumlichen Daten gerendert werden
  • Web Map Tile Service (WMTS): Liefert vorberechnete Kartenkacheln.
  • Catalogue Service for the Web (CSW): Durchführung von Suchen nach räumlichen Datensätzen und Diensten.
  • Web Processing Service (WPS): Führt Berechnungen auf räumlichen Daten aus.

Eigenschaften des deegree WFS¶

Der deegree WFS ist eine Implementierung der OGC Web Feature Service Spezifikation. Einige Eigenschaften:

  • Implementiert WFS standards 1.0.0, 1.1.0 and 2.0.0
  • Vollständig transaktional (auch bei komplexen Datenmodellen)
  • Unterstützt KVP, XML und SOAP Requests
  • GML 2/3.0/3.1/3.2 Output/Input
  • Unterstützung von GetGmlObject Requests und XLinks
  • Hoch performant und hervorragend skalierbar
  • Koordinatentransformation on-the-fly
  • Von Grund auf für komplexe Datenmodelle entworfen
  • Die Back-Ends unterstützen eine flexible Abbildung von GML-Applikationsschemata in relationale Modelle
  • ISO 19107 konformes geometrisches Modell: Komplexe Geometrie (z.B. nicht lineare Kurven)
  • Unterstützt fortschrittliche Filterausdrücke basierend auf XPath 1.0
  • Unterstützt eine Vielzahl von Datenquellen, wie beispielsweise PostGIS, Oracle Stpatial, MS SQL Server, Shapefile oder GML-Dokumente.

Der deegree WMS ist eine Implementierung der OGC Web Map Service Spezifikation. Einige Eigenschaften:

  • Implementiert die WMS Standards 1.1.1 und 1.3.0
  • Extensive support for styling languages SLD/SE versions 1.0.0 and 1.1.0
  • Hoch performant und hervorragend skalierbar
  • Hoch qualitatives Rendering
  • Maßstabsabhängige Signaturen
  • Support for SE removes the need for a lot of proprietary extensions
  • Easy configuration of HTML and other output formats for GetFeatureInfo responses
  • Uses stream-based data access with minimal memory footprint
  • Nearly complete support for raster symbolizing as defined in SE (with some extensions)
  • Complete support for TIME/ELEVATION and other dimensions for both feature and raster data
  • Supports numerous backends, such as PostGIS, Oracle Spatial, Shapefiles or GML instance documents
  • Can render rich data models directly

Der deegree WMTS ist eine Implementierung der OGC Web Map Tile Service Spezifikation. Einige Eigenschaften:

  • Implements Basic WMTS standard 1.0.0 (KVP)
  • Hoch performant und hervorragend skalierbar
  • Unterstützt verschiedene Datenquellen, wie beispielsweise GeoTIFF, entfernte WMS oder hierarchische Rasterkacheln im Dateisystem
  • Unterstützt on-the-fly Caching (Verwendung von EHCache)
  • Unterstützt GetFeatureInfo für entfernte WMS

Der deegree CSW ist eine Implementierung der OGC Catalogue Service Spezifikation. Einige Eigenschaften:

  • Implementiert CSV standard 2.0.2
  • Vollständig transaktional
  • Unterstützt KVP, XML und SOAP Requests
  • Hoch performant und hervorragend skalierbar
  • ISO Metadata Application Profile 1.0.0
  • Pluggable and modular dataaccess layer allows to add support for new APs and backends
  • Modular inspector architecture allows to validate records to be inserted against various criteria
  • Standard inspectors: schema validity, identifier integrity, INSPIRE requirements
  • Handles all defined queryable properties (for Dublin Core as well as ISO profile)
  • Complex filter expressions

Der deegree WPS ist eine Implementierung der OGC Processing Service Spezifikation. Einige Eigenschaften:

  • Implements WPS standard 1.0.0
  • Unterstützt KVP, XML und SOAP Requests
  • Pluggable process provider layer
  • Easy-to-use API for implementing Java processes
  • Supports all variants of input/output parameters: literal, bbox, complex (binary and xml)
  • Streaming access for complex input/output parameters
  • Processing of huge amounts of data with minimal memory footprint
  • Supports storing of response documents/output parameters
  • Supports input parameters given inline and by reference
  • Supports RawDataOutput/ResponseDocument responses
  • Supports asynchronous execution (with polling of process status)

Implementierte Standards¶

  • OGC Web Feature Service (WFS) 1.0.0, 1.1.0 und 2.0.0
  • OGC Web Map Service (WMS) 1.1.1 und 1.3.0
  • OGC Web Map Tile Service (WMTS) 1.1.0
  • OGC Catalogue Service-Web (CSW) 2.0.2
  • OGC Web Processing Service (WPS) 1.0.0
  • OGC GML 2.12, 3.0.1, 3.1.1 und 3.2.1
